SaaS Boilerplate Version Context
Version: v0.6.0
Name: RBAC Foundation
Date: 2026-05-21

Purpose
-------
This version adds the RBAC foundation to the organisation-first SaaS boilerplate.
It builds on v0.5.0 native authentication and starts protecting internal APIs using bearer user session tokens and permission checks.

Locked Design Decisions
-----------------------
1. MongoDB remains the main software database.
2. Tenant model remains Organisation -> Users.
3. A user belongs to exactly one organisation.
4. A user cannot exist without organisation_id.
5. Every tenant-owned document must include organisation_id.
6. Normal API requests must not trust organisation_id from request bodies.
7. Current protected access uses logged-in user access tokens.
8. Future token strategy includes Personal Access Tokens, Organisation API Tokens and Service Account Tokens.
9. Every API must include Swagger/OpenAPI documentation, Internal/External tags and authentication/permission notes.

RBAC Rules
----------
- /api/permissions requires permissions.view.
- /api/org/roles list/view requires roles.view.
- /api/org/roles create/update/delete requires roles.manage.
- /api/internal/schema/status requires internal.schema.view.
- /api/platform/organisations view APIs require platform.organisations.view.
- /api/platform/organisations create/update/status APIs require platform.organisations.manage.
- /api/platform/organisations/:organisationId/users APIs require users.* permissions and organisation parameter access.

Seed Updates
------------
The seed script now creates these additional permissions:
- platform.organisations.view
- platform.organisations.manage
- internal.schema.view
- permissions.view
- users.update

The Organisation Admin role receives all seeded permissions.
The Viewer role remains read-only and limited.
