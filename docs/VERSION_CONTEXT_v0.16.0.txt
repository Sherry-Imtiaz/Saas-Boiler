SaaS Boilerplate Version Context
================================

Version: v0.16.0
Build Name: Testing Foundations
Canonical Previous Version: v0.15.0 - Deployment Readiness

Purpose
-------
v0.16.0 adds the formal testing foundation for the SaaS Boilerplate. It introduces a repeatable API test stack, fast unit tests, integration test helpers, isolated test database support and documentation for local and CI-style validation.

What v0.16.0 Adds
-----------------
- Vitest API test runner configuration.
- Supertest API integration test support.
- mongodb-memory-server support for isolated integration tests.
- TEST_MONGODB_URI override for machines that prefer a local Docker MongoDB test database.
- apps/api/.env.test.example.
- Reusable test setup and test data helpers.
- Unit tests for password hashing/verification.
- Unit tests for JWT signing/verification.
- Integration tests for health liveness/readiness endpoints.
- Integration tests for native login success and failed-login security events.
- Integration tests for organisation audit log tenant scoping.
- Integration tests for platform audit log visibility.
- Integration tests for audit permission enforcement.
- Integration tests for organisation security event tenant scoping.
- Integration tests for security event filtering.
- Integration tests for platform security event visibility.
- Testing documentation in docs/TESTING_FOUNDATIONS.md.
- Runtime version metadata updated to v0.16.0.
- Static OpenAPI JSON/YAML files for v0.16.0.

Important Design Rules Preserved
--------------------------------
- Organisation -> Users.
- A user belongs to one organisation only.
- No user can exist without an organisation.
- Swagger/OpenAPI remains domain grouped.
- Internal/external classification remains endpoint metadata.
- User session tokens are used for internal app access.
- Personal access tokens and organisation API tokens remain scoped, auditable, revocable and organisation-bound.
- Compute allocation and compute jobs remain optional future product modules, not part of the minimum boilerplate core.

Validation Notes
----------------
- Fast unit tests are the default root npm test target.
- Integration tests require either mongodb-memory-server binary download access or TEST_MONGODB_URI pointing to a local test MongoDB instance.
- External OIDC live-provider tests are not included in v0.16.0 because they require real IdP credentials and callback configuration.
