Core architecture rule
- Organisation -> Users.
- A user belongs to exactly one organisation.
- A user cannot exist without an organisation.
- Every tenant-owned record must be scoped to organisation_id.
- Normal user-facing APIs should resolve organisation_id from the authenticated token, not from the frontend.

Authentication and permissions
- All plan APIs are tagged Internal.
- Plan read APIs require UserSessionAuth or PersonalAccessTokenAuth and plans.view.
- Plan write/assignment APIs require UserSessionAuth or PersonalAccessTokenAuth and plans.manage.

Plan catalogue defaults
- starter: basic limits, file uploads enabled, compute disabled.
- professional: larger limits, external API access enabled, compute allocation ready but compute jobs disabled.
- enterprise: larger limits, compute jobs, reporting and external APIs enabled.
- custom: flexible/null limits for manually negotiated tenants.

Operational notes
- Future file upload and compute builds should read plan limits before accepting resource usage.
- Feature entitlement checks still remain separate from RBAC permission checks.
- A plan can define feature defaults, but organisation feature entitlements remain stored on the organisation.
- Applying a plan can optionally apply plan feature defaults to the organisation.
