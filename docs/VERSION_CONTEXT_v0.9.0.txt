Feature Access Rule
- A feature being enabled does not grant access by itself.
- A user/token still needs RBAC permission or token scope.
- Future protected module routes should check both feature and permission.

Example
- Organisation feature compute_jobs = true.
- User permission compute.jobs.create = true.
- Result: user can create compute jobs after the compute jobs module exists.

If compute_jobs = false, the user must still be blocked even if they have compute.jobs.create.

New APIs
1. GET /api/features/catalogue
   Tag: Internal
   Auth: UserSessionAuth or PersonalAccessTokenAuth
   Permission: features.view
   Purpose: Return global feature catalogue.

2. GET /api/org/features
   Tag: Internal
   Auth: UserSessionAuth or PersonalAccessTokenAuth
   Permission: features.view
   Purpose: Return feature entitlement state for the authenticated organisation.

3. PATCH /api/org/features
   Tag: Internal
   Auth: UserSessionAuth or PersonalAccessTokenAuth
   Permission: features.manage
   Purpose: Enable/disable organisation feature flags.
