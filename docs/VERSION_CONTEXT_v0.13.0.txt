Current architecture rule
-------------------------
Organisation -> Users.
A user belongs to exactly one organisation.
A user cannot exist without an organisation.
Tenant-owned data must remain organisation-scoped.

What v0.13.0 adds
-----------------
- Refactored Swagger/OpenAPI tags into functional/domain-based groups.
- Removed Internal/External as primary Swagger grouping categories.
- Retained internal/public/external classification as endpoint metadata using x-saas-classification.
- Added endpoint metadata for required permission, tenant scope and accepted token types.
- Added stable machine-readable API error codes.
- Updated the shared HttpError utility with default code mapping.
- Updated the global error handler to emit error.code consistently.
- Added requireAllPermissions() for sensitive routes requiring more than one permission.
- Hardened platform organisation plan routes so they require both platform organisation permissions and plan permissions.
- Improved permission/scope failure messages to avoid leaking unnecessary internal detail in message text.
- Fixed local OIDC_STATE_SECRET handling so empty local development values no longer fail seed/start.
- Added docs/API_SECURITY_HARDENING.md.
- Added static OpenAPI JSON/YAML for v0.13.0.

Security-sensitive route hardening
----------------------------------
The following routes now require combined permissions:
- GET /api/platform/organisations/:organisationId/plan
  - platform.organisations.view
  - plans.view
- PATCH /api/platform/organisations/:organisationId/plan
  - platform.organisations.manage
  - plans.manage
- POST /api/platform/organisations/:organisationId/plan/apply-defaults
  - platform.organisations.manage
  - plans.manage

Internal/external classification rule
-------------------------------------
Internal, external and public classification is no longer the main Swagger grouping.
It is captured as endpoint metadata:
- x-saas-classification
- x-required-permission
- x-required-scope
- x-tenant-scope
- x-token-types
